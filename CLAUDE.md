# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

**RRWrite** is an AI-powered tool that transforms research code repositories into publication-ready scientific manuscripts. It analyzes repositories, conducts literature reviews, generates structured academic writing, and provides adversarial critique with automated revision capabilities.

## Key Architecture

### Dual Git Repository System

**Critical:** RRWrite maintains separate Git repositories for the tool and manuscripts:

- **Tool Repository** (`rrwrite/.git/`): This repository containing scripts and skills
- **Manuscript Repositories** (`manuscript/project_v1/.git/`): Each manuscript gets its own `.git/`

The `manuscript/` directory is gitignored by the tool repo. Four safety layers prevent manuscript commits polluting the tool repo:
1. `.gitignore` excludes `manuscript/`
2. Pre-commit hooks reject manuscript files
3. `GitManager` class validates remote URLs
4. `StateManager` checks for tool repo markers

**See:** `docs/GIT_ARCHITECTURE.md` for complete explanation.

### State Management System

Workflow state tracked in `{manuscript_dir}/.rrwrite/state.json`:

```python
# State machine: not_started → in_progress → completed
workflow_stages = [
    "repository_analysis",      # Analyze code structure
    "plan",                     # Generate outline
    "assessment",               # Match to journal
    "research",                 # Literature review
    "figure_table_extraction",  # Extract figures/tables from repo (NEW!)
    "drafting",                 # Write sections (6 sub-stages)
    "assembly",                 # Combine sections
    "critique"                  # Adversarial review
]
```

**Key files:**
- `scripts/rrwrite_state_manager.py` - StateManager class
- `scripts/rrwrite_git.py` - GitManager with safety checks

### Evidence-Based Writing

All manuscript claims must link to repository evidence:

```
Repository → Evidence Extraction → Citation Index → Manuscript → Verification
```

**Key scripts:**
- `scripts/rrwrite-extract-repo-evidence.py` - Extract facts from repo
- `scripts/rrwrite-verify-evidence.py` - Verify claims against evidence
- `scripts/rrwrite-generate-evidence-report.py` - Generate EVIDENCE_REPORT.md

### Figure and Table Extraction System

**Priority-based extraction** that distinguishes between actual research outputs and analysis visualizations:

**Priority 1 (Repository Figures/Tables):**
- Extracted from analyzed repository (`figures/from_repo/`, `tables/from_repo/`)
- Actual research outputs generated by the code being documented
- Takes precedence in section drafting

**Priority 2 (Generated Visualizations):**
- Created from repository analysis data (`figures/generated/`, `tables/generated/`)
- Supplementary analysis visualizations (composition charts, file distributions)

**Key features:**
- Automatic script detection (links figures to generating code)
- Smart filtering (excludes thumbnails, icons, oversized files)
- JSON manifests with section recommendations
- README index generation for discoverability

**Key scripts:**
- `scripts/rrwrite-extract-figures-tables.py` - Main extraction tool
- `scripts/rrwrite_manifest_generator.py` - Manifest creation/validation
- `scripts/rrwrite_figure_generator.py` - Enhanced with manifest queries

### Automated Revision System

After critique, manuscripts can be automatically revised with convergence detection:

- **Parser**: `rrwrite_revision_parser.py` - Extracts issues from critique reports
- **Context**: `rrwrite_revision_context.py` - Loads evidence/citations for section revisers
- **Revisers**: `rrwrite_section_reviser.py` - Section-specific revision strategies
- **Orchestrator**: `rrwrite-revise-manuscript.py` - Iterative improvement loop

Revision stops when:
- Issues fully addressed
- Improvement rate < 5%
- Max iterations reached (default: 2)

### Edit Recommendation System

**NEW!** Transform critique and external feedback into structured, actionable edit recommendations:

- **EditRecommendation** (`rrwrite_edit_recommendation.py`) - Structured edit with priority/metadata
- **EditRecommendationGenerator** (`rrwrite_edit_recommendation_generator.py`) - Converts issues → edits
- **ExternalFeedbackParser** (`rrwrite_external_feedback_parser.py`) - Parses Word/PDF/email feedback
- **CLI**: `rrwrite-generate-edit-recommendations.py` - Generate recommendations
- **Validation**: `rrwrite-validate-edit-recommendations.py` - Validate recommendations

**Priority System:**
- **Critical**: Blocks publication (reproducibility, major evidence gaps)
- **Important**: Significantly improves manuscript (major issues, formatting)
- **Optional**: Minor improvements

**Edit Types:**
- add_content, remove_content, revise_content, restructure
- citation_fix, figure_update, table_update, move_content

**Usage:**
```bash

    --manuscript-dir manuscript/project_v1 --version 1

# From Word comments
python scripts/rrwrite-generate-edit-recommendations.py \
    --manuscript-dir manuscript/project_v1 \
    --word-comments reviewer_feedback.docx

# Combine sources
python scripts/rrwrite-generate-edit-recommendations.py \
    --manuscript-dir manuscript/project_v1 \
    --version 1 --word-comments feedback.docx
```

**Outputs:**
- `edit_recommendations_v1.json` - Structured recommendations
- `edit_recommendations_v1.md` - Human-readable summary

### Holistic Edit Application

**NEW!** Apply edit recommendations cohesively across entire manuscript with dependency analysis:

- **HolisticEditOrchestrator** (`rrwrite_holistic_editor.py`) - Main coordinator
- **DependencyGraph** - Topological sort for edit order
- **ApplicationPlan** - Conflict detection and resolution
- **Specialized Applicators** (`rrwrite_edit_applicators.py`):
  - SectionEditApplicator - Text edits (add/remove/revise)
  - CrossSectionApplicator - Move content between sections
  - FigureEditApplicator - Update captions and references
  - TableEditApplicator - Update titles and content
  - ConsistencyApplicator - Standardize terminology, numbering
- **ConsistencyChecker** (`rrwrite_consistency_checker.py`) - Detect inconsistencies
- **CLI**: `rrwrite-apply-edits.py` - Apply edits holistically
- **CLI**: `rrwrite-check-consistency.py` - Check consistency

**Architecture:**
```
Edit Ingestion → Dependency Analysis → Conflict Detection →
Transaction Begin → Apply Edits → Validate → Commit/Rollback
```

**Usage:**
```bash
# Preview edits (dry run)
python scripts/rrwrite-apply-edits.py \
    --manuscript-dir manuscript/project_v1 \
    --recommendations edit_recommendations_v1.json \
    --dry-run

# Apply critical edits only
python scripts/rrwrite-apply-edits.py \
    --manuscript-dir manuscript/project_v1 \
    --recommendations edit_recommendations_v1.json \
    --priority critical

# Apply all edits with backup (default)
python scripts/rrwrite-apply-edits.py \
    --manuscript-dir manuscript/project_v1 \
    --recommendations edit_recommendations_v1.json

# Check consistency
python scripts/rrwrite-check-consistency.py \
    --manuscript-dir manuscript/project_v1
```

**Safety Features:**
- Automatic dependency resolution (topological sort)
- Conflict detection (priority-based resolution)
- Transaction-based application (backup/rollback)
- Fuzzy matching for content location (85% threshold)

**Outputs:**
- Updated section files
- Updated figure/table manifests
- `edit_application_report_v1.md` - Application results
- `consistency_report_v1.md` - Consistency issues

### Citation Gap Analysis for Google Docs

**NEW!** Identify missing citations in Google Docs manuscripts with semantic similarity analysis:

- **Download Tool** (`rrwrite-download-gdoc.py`) - Export Google Doc as DOCX
- **Citation Extractor** (`rrwrite-extract-gdoc-citations.py`) - Parse Paperpile/bracketed/numbered citations
- **Gap Analyzer** (`rrwrite-citation-gap-analyzer.py`) - 4-layer gap analysis with TF-IDF
- **Report Generator** (`rrwrite-generate-gap-report.py`) - Actionable Markdown + JSON reports
- **Workflow Orchestrator** (`rrwrite-gdoc-citation-gap-workflow.py`) - End-to-end automation

**4-Layer Analysis:**
1. **Exact Matching** - DOI/title comparison
2. **Semantic Overlap** - TF-IDF cosine similarity (threshold: 70%)
3. **Citation Type Categorization** - tool, method, review, benchmark, dataset
4. **Citation Network Analysis** - Impact-based prioritization

**Multi-Tier Literature Search:**
- **Tier 1:** Direct method match (foundational, highly-cited >50)
- **Tier 2:** Overlapping approaches (recent, specialized)
- **Tier 3:** Related infrastructure (FAIR, provenance, reproducibility)

**Usage:**
```bash
# Quick start (end-to-end workflow)
python scripts/rrwrite-gdoc-citation-gap-workflow.py \
    --document-id 1ABC...XYZ \
    --output-dir manuscript/citation_gaps \
    --search-tiers 1,2,3 \
    --credentials credentials.json

# Step-by-step workflow
python scripts/rrwrite-download-gdoc.py \
    --document-id 1ABC...XYZ \
    --output manuscript.docx \
    --credentials credentials.json

python scripts/rrwrite-extract-gdoc-citations.py \
    --docx manuscript.docx \
    --output extracted_citations.json \
    --stats

python scripts/rrwrite-citation-gap-analyzer.py \
    --manuscript-citations extracted_citations.json \
    --search-results tier1_results.json tier2_results.json \
    --output gap_analysis.json \
    --similarity-threshold 0.70

python scripts/rrwrite-generate-gap-report.py \
    --gap-analysis gap_analysis.json \
    --output-dir manuscript/citation_gaps
```

**Prioritization:**
- **Critical** (score ≥7): Must-cite papers (>100 citations, >70% overlap)
- **Important** (score ≥4): Should-cite papers (strengthen manuscript)
- **Optional** (score <4): Emerging work or niche topics

**Output:**
- `citation_gap_report.md` - Human-readable actionable recommendations
- `citation_gap_report.json` - Machine-readable structured data
- Section-specific citation suggestions with context

**Runtime:** 5-15 minutes (first run), 30-60 seconds (cached)

**Prerequisites:**
- Google Docs API + Drive API enabled
- OAuth2 credentials or Service Account
- python-docx, scikit-learn (optional but recommended)

**See:**
- `CITATION_GAP_QUICK_START.md` - 5-minute quick start guide
- `docs/CITATION_GAP_ANALYSIS.md` - Comprehensive documentation
- `docs/GDOC_API_SETUP.md` - Google API setup guide

## Architecture Details

### StateManager Class

**Location:** `scripts/rrwrite_state_manager.py`

Manages workflow state and Git initialization:

```python
StateManager(output_dir="manuscript/project_v1", enable_git=True, auto_commit=True)

# Key methods:
.start_stage(stage_name)           # Mark stage as in_progress
.complete_stage(stage_name, data)  # Mark completed, auto-commit
.get_stage_status(stage_name)      # Get current status
.get_progress_percentage()         # Overall completion
```

**State file:** `{manuscript_dir}/.rrwrite/state.json`

### GitManager Class

**Location:** `scripts/rrwrite_git.py`

Handles Git operations with safety checks:

```python
GitManager(manuscript_dir, auto_commit=True)

# Safety validation:
- Checks not operating on tool repo
- Validates manuscript directory has .rrwrite/
- Explicit --git-dir usage for all git commands
- Remote URL validation

# Key methods:
.initialize_repository()   # Create new git repo
.commit(message, files)    # Commit with safety checks
.get_commit_history()      # Query history
```

### Revision System

**Location:** `scripts/rrwrite-revise-manuscript.py` and supporting modules

Three-component architecture:

1. **CritiqueParser** (`rrwrite_revision_parser.py`)
   - Parses critique_content_v{N}.md and critique_format_v{N}.md
   - Extracts issues with severity, section, and description
   - Infers section from issue context

2. **RevisionContext** (`rrwrite_revision_context.py`)
   - Loads evidence database, citations, journal guidelines
   - Provides context for section revisers
   - Caches loaded data

3. **SectionRevisers** (`rrwrite_section_reviser.py`)
   - Section-specific revision strategies
   - Available revisers: abstract, introduction, methods, results, discussion, availability
   - Each reviser knows section constraints (word limits, required elements)

4. **RevisionOrchestrator** (`rrwrite-revise-manuscript.py`)
   - Main iteration loop: parse → revise → validate → critique → converge
   - Convergence detection based on improvement rate
   - Automatic re-critique after each iteration

### Citation and Evidence Systems

**Citation Management:**
- Search: `rrwrite-search-literature.py` (PubMed + Semantic Scholar)
- Database: `literature_evidence.csv` (title, authors, year, doi, abstract)
- Bibliography: `literature_citations.bib` (BibTeX format)
- Validation: `rrwrite_citation_validator.py` (DOI verification, duplicate detection)

**Evidence Tracking:**
- Extract: `rrwrite-extract-repo-evidence.py` (README, docstrings, comments, tests)
- Database: `repo_evidence.csv` (claim_id, claim_text, source_file, line_number)
- Verify: `rrwrite-verify-evidence.py` (check manuscript claims link to evidence)
- Report: `EVIDENCE_REPORT.md` (summary of evidence usage)

## Safety Considerations

### Git Repository Separation

**Always work from rrwrite root, never commit manuscripts to tool repo:**

```bash
# CORRECT: Commit manuscript changes to manuscript repo
cd manuscript/project_v1
git add sections/introduction.md
git commit -m "Update introduction"

# WRONG: Trying to commit manuscripts to tool repo (will be rejected by pre-commit hook)
cd /path/to/rrwrite
git add manuscript/project_v1/sections/introduction.md  # ❌ Pre-commit hook prevents this
```

### State File Location

State is always at `{manuscript_dir}/.rrwrite/state.json`, not in tool directory.

### Evidence Verification

Before assembly, verify all claims have evidence:
```bash
python scripts/rrwrite-verify-evidence.py \
    --manuscript manuscript/project_v1/manuscript.md \
    --evidence manuscript/project_v1/repo_evidence.csv
```

Missing evidence = unsubstantiated claims that will be flagged in critique.

## Troubleshooting

### "SAFETY VIOLATION: RRWRITE TOOL REPOSITORY DETECTED"

**Cause:** Trying to initialize Git in tool directory
**Fix:** Use `manuscript/` subdirectory

```bash
# Correct
/rrwrite --repo /path/to/repo --output-dir manuscript/project_v1

# Wrong
/rrwrite --repo /path/to/repo --output-dir .
```

### Pre-commit Hook Rejects Manuscript Files

**Cause:** Attempting to commit manuscript/ files to tool repo
**Fix:** Commit from manuscript directory instead

```bash
git reset HEAD manuscript/
cd manuscript/project_v1
git add .
git commit -m "Manuscript changes"
```

### Missing Citations or Evidence

**Cause:** Literature search incomplete or evidence extraction failed
**Fix:** Re-run search/extraction

```bash
python scripts/rrwrite-search-literature.py --output-dir manuscript/project_v1 --topics "topic1,topic2"
python scripts/rrwrite-extract-repo-evidence.py --repo-path /path/to/repo --output manuscript/project_v1/repo_evidence.csv
```

### Assembly Validation Failures

**Cause:** Incomplete sections or citation mismatches
**Fix:** Check validation report

```bash
python scripts/rrwrite-validate-manuscript.py --manuscript-dir manuscript/project_v1
# Reads: assembly_manifest.json
# Checks: section completeness, citation consistency, word counts
```

### Revision Not Converging

**Cause:** Issues persist or improvement too slow
**Fix:** Manually revise problematic sections

```bash
# Check which issues remain
cat manuscript/project_v1/critique_content_v2.md | grep "MAJOR:"

# Manually fix
vim manuscript/project_v1/sections/methods.md

# Re-assemble and critique
python scripts/rrwrite-assemble-manuscript.py --manuscript-dir manuscript/project_v1
/rrwrite-critique-manuscript
```

## Key Concepts

- **Manuscript** = Complete output directory with sections, evidence, state
- **Stage** = Workflow phase (analysis, planning, drafting, etc.)
- **Evidence** = Facts extracted from repository that support manuscript claims
- **Critique** = Adversarial review identifying content and format issues
- **Revision** = Automated improvement based on critique reports
- **State** = JSON file tracking workflow progress and metadata
- **Assembly** = Combining section files into complete manuscript with citation formatting
