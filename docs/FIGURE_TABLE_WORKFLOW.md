# Figure and Table Extraction Workflow Guide

**Version:** 1.0
**Date:** 2026-03-05
**Stage:** Phase 5 (after Literature Research, before Section Drafting)

## Overview

The Figure and Table Extraction system automatically discovers and organizes visual content from analyzed repositories, creating a **priority-based system** that distinguishes between actual research outputs and supplementary analysis visualizations.

## Priority System

### Priority 1: Repository Figures/Tables (Actual Research Outputs)

**Source:** Extracted from the analyzed code repository
**Location:** `figures/from_repo/`, `tables/from_repo/`
**Purpose:** Actual research outputs generated by the code being documented

These are the **primary visual evidence** of your research. They represent:
- Experimental results generated by your code
- Data visualizations from your analysis
- Workflow diagrams showing your methodology
- Performance comparisons and benchmarks

**Example:** If documenting a machine learning pipeline, Priority 1 figures would include:
- Model accuracy plots generated by `train_model.py`
- Confusion matrices from `evaluate.py`
- Feature importance visualizations
- ROC curves from cross-validation

### Priority 2: Generated Visualizations (Supplementary Analysis)

**Source:** Created from repository analysis data
**Location:** `figures/generated/`, `tables/generated/`
**Purpose:** Supplementary visualizations about the repository itself

These are **metadata visualizations** that help readers understand your codebase:
- Repository composition (file types, sizes)
- Code complexity metrics
- Research topic radar charts
- File size distributions

**Example:** For the same ML pipeline repository:
- Bar chart showing distribution of Python vs Jupyter files
- Pie chart of repository composition
- File size distribution histogram

## When to Run Extraction

**Position in workflow:**
```
Repository Analysis → Planning → Assessment → Literature Research
    → Figure/Table Extraction ← YOU ARE HERE
    → Section Drafting → Assembly → Critique
```

**Prerequisites:**
- ✓ Repository analysis completed (`data_tables/` directory exists)
- ✓ Repository path is known and accessible

**Timing:** 2-4 minutes for typical repositories

## Running the Extraction

### Method 1: Claude Skill (Recommended)

```bash
/rrwrite-extract-figures-tables \
    --target_dir manuscript/project_v1 \
    --repo_path /path/to/analyzed/repo
```

### Method 2: Python Script

```bash
python scripts/rrwrite-extract-figures-tables.py \
    --repo-path /path/to/analyzed/repo \
    --manuscript-dir manuscript/project_v1 \
    --extract figures,tables \
    --generate figures \
    --verbose
```

### Method 3: Custom Options

```bash
# Extract only figures (skip tables)
python scripts/rrwrite-extract-figures-tables.py \
    --repo-path /path/to/repo \
    --manuscript-dir manuscript/project_v1 \
    --extract figures \
    --generate figures

# Extract only tables (skip figures)
python scripts/rrwrite-extract-figures-tables.py \
    --repo-path /path/to/repo \
    --manuscript-dir manuscript/project_v1 \
    --extract tables \
    --generate tables

# Skip generation (only extract from repo)
python scripts/rrwrite-extract-figures-tables.py \
    --repo-path /path/to/repo \
    --manuscript-dir manuscript/project_v1 \
    --extract figures,tables \
    --generate ""
```

## Understanding Manifests

### Figure Manifest Structure

```json
{
  "version": "1.0",
  "created_at": "2025-01-15T10:30:00",
  "total_figures": 8,
  "figures_from_repo": 5,
  "figures_generated": 3,
  "figures": [
    {
      "id": "fig_repo_001",
      "path": "figures/from_repo/workflow_diagram.pdf",
      "source": "from_repo",
      "priority": 1,
      "original_path": "docs/figures/workflow.pdf",
      "recommended_sections": ["methods", "introduction"],
      "default_caption": "Workflow diagram showing analysis pipeline",
      "generating_script": "scripts/create_workflow.py"
    }
  ]
}
```

**Key fields:**
- `id`: Unique identifier (e.g., `fig_repo_001`, `fig_gen_001`)
- `priority`: 1 = from_repo (use first), 2 = generated (supplementary)
- `recommended_sections`: Where this figure is most relevant
- `generating_script`: Python/R script that created this figure (if detected)

## Using Extracted Figures in Sections

### Discovery During Drafting

When drafting a section (e.g., Results), query the manifest to find available figures:

```python
from pathlib import Path
from scripts.rrwrite_figure_generator import FigureSelector

# Get figures for results section (prioritizes repo figures)
figures = FigureSelector.get_figures_from_manifest(
    section_name="results",
    manifest_path=Path("manuscript/project_v1/figures/figure_manifest.json"),
    prioritize_repo_figures=True
)

for fig in figures:
    priority = "REPO" if fig['priority'] == 1 else "GENERATED"
    print(f"[{priority}] {fig['id']}: {fig['default_caption']}")
    print(f"  Path: {fig['path']}")
    if fig.get('generating_script'):
        print(f"  Script: {fig['generating_script']}")
```

**Output:**
```
[REPO] fig_repo_003: Model accuracy comparison across datasets
  Path: figures/from_repo/accuracy_comparison.png
  Script: experiments/plot_results.py
[REPO] fig_repo_007: Confusion matrix for test set
  Path: figures/from_repo/confusion_matrix.pdf
  Script: experiments/evaluate_model.py
[GENERATED] fig_gen_001: Repository composition by file category
  Path: figures/generated/repository_composition.png
```

### Including in Markdown

**Standard format:**
```markdown
![Model accuracy comparison](../figures/from_repo/accuracy_comparison.png)

**Figure 3**: Model accuracy comparison across three benchmark datasets.
The proposed approach (red) consistently outperforms baseline methods
(blue) with p < 0.001 for all comparisons (paired t-test, n=100 runs).
```

**Best practices:**
- Use relative paths from `sections/` directory: `../figures/from_repo/`
- Caption format: `**Figure N**: Description`
- Reference in text: "As shown in Figure 3, our method..."
- Prioritize repository figures over generated when both available

## Smart Filtering

The extraction system automatically **excludes**:

### File Patterns
- Thumbnails: `*thumb*`, `*thumbnail*`
- Icons: `*icon*`, `*logo*`, `*badge*`
- Temporary files: `*tmp*`, `*temp*`

### Directory Patterns
- Build artifacts: `build/`, `dist/`, `target/`
- Dependencies: `node_modules/`, `vendor/`
- Version control: `.git/`, `.svn/`
- Cache: `__pycache__/`, `.pytest_cache/`, `.ipynb_checkpoints/`

### Size Limits
- **Figures:** Max 10 MB (prevents copying large intermediate files)
- **Tables:** Max 5 MB (prevents copying raw data dumps)

## Script Detection

The extractor attempts to link figures to their generating scripts by:

1. Searching the same directory for `.py`, `.R`, `.r`, `.ipynb` files
2. Looking for filename references in script content
3. Storing the relative path in `generating_script` field

**Example:**
```python
# In experiments/plot_results.py:
plt.savefig('figures/accuracy_comparison.png')

# Extracted metadata:
{
  "id": "fig_repo_003",
  "path": "figures/from_repo/accuracy_comparison.png",
  "generating_script": "experiments/plot_results.py"  # ← Detected!
}
```

This helps readers:
- Understand how figures were generated
- Reproduce results by running the script
- Verify figure authenticity

## Troubleshooting

### No Figures Found

**Symptom:** `Found 0 figures (filtered from 0)`

**Causes:**
- Repository has no figure files
- All figures filtered by exclusion patterns
- Wrong repository path

**Solutions:**
```bash
# Manual check for figures
find /path/to/repo -name "*.png" -o -name "*.pdf" -o -name "*.svg" | \
  grep -v ".git" | head -20

# Check if filtering too aggressive
python scripts/rrwrite-extract-figures-tables.py \
    --repo-path /path/to/repo \
    --manuscript-dir manuscript/test \
    --extract figures \
    --verbose
```

### Manifests Not Created

**Symptom:** `figure_manifest.json` missing

**Causes:**
- No figures/tables extracted
- Permission errors

**Solutions:**
```bash
# Check directory permissions
ls -la manuscript/project_v1/figures/

# Re-run with verbose output
python scripts/rrwrite-extract-figures-tables.py \
    --repo-path /path/to/repo \
    --manuscript-dir manuscript/project_v1 \
    --extract figures \
    --verbose
```

### Figures Not Embedding in DOCX

**Symptom:** Broken image links in Word document

**Causes:**
- Incorrect relative paths in markdown
- Missing resource paths in Pandoc

**Solutions:**
```bash
# Verify paths in sections
grep -r "!\[.*\]" manuscript/project_v1/sections/

# Check assembly script resource paths
grep "resource_paths" scripts/rrwrite-assemble-manuscript.py

# Re-assemble
python scripts/rrwrite-assemble-manuscript.py \
    --output-dir manuscript/project_v1
```

### Too Many Figures Extracted

**Symptom:** 500+ figures extracted (mostly duplicates/PDFs)

**Causes:**
- Repository contains downloaded papers (PDFs)
- Multiple versions of same figure

**Solutions:**
- Manually curate `figures/from_repo/` directory
- Update exclusion patterns if specific patterns detected
- Use `--extract ""` to skip extraction, generate only

## Best Practices

### 1. Prioritize Repository Figures

When both repository and generated figures are available:
- **Use repository figures** for actual research results
- **Use generated figures** for repository metadata/composition

### 2. Verify Figure Authenticity

Before including a figure in your manuscript:
- Check `generating_script` to understand how it was created
- Run the script to verify reproducibility
- Ensure figure shows what you claim it shows

### 3. Curate Figure Selection

Not all extracted figures belong in the manuscript:
- Review `figures/from_repo/README.md` for discovered figures
- Select only the most relevant for each section
- Ensure figures support your narrative

### 4. Write Descriptive Captions

Default captions are generated from filenames, but you should:
- Expand with method details (e.g., sample size, statistical tests)
- Explain what patterns readers should notice
- Link to relevant Methods section for generation details

### 5. Maintain Traceability

Always preserve the link between figures and generating code:
- Reference `generating_script` in Methods section
- Ensure scripts are in repository with clear documentation
- Include version/commit information if figures may change

## Advanced Usage

### Custom Manifest Queries

```python
from pathlib import Path
from scripts.rrwrite_manifest_generator import ManifestGenerator

generator = ManifestGenerator(Path("manuscript/project_v1"))

# Get all Priority 1 figures
manifest = generator.load_figure_manifest()
repo_figures = [f for f in manifest['figures'] if f['priority'] == 1]

# Get figures by source file
workflow_figs = [
    f for f in manifest['figures']
    if 'workflow' in f.get('generating_script', '').lower()
]

# Get figures without generating script
unknown_figs = [
    f for f in manifest['figures']
    if not f.get('generating_script')
]
```

### Programmatic Figure Selection

```python
def select_key_figures(manifest_path, max_figures=6):
    """Select most important figures for manuscript."""
    with open(manifest_path) as f:
        manifest = json.load(f)

    # Prioritize: repo figures > generated figures
    # Prioritize: results section > other sections
    scored_figures = []
    for fig in manifest['figures']:
        score = 0
        score += 10 if fig['priority'] == 1 else 0
        score += 5 if 'results' in fig['recommended_sections'] else 0
        score += 3 if fig.get('generating_script') else 0
        scored_figures.append((score, fig))

    # Sort by score and return top N
    scored_figures.sort(reverse=True, key=lambda x: x[0])
    return [fig for _, fig in scored_figures[:max_figures]]
```
