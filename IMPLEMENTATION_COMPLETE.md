## 🎯 Key Features Delivered

### 1. Priority-Based Organization

```
figures/
├── from_repo/     # Priority 1: Research outputs
└── generated/     # Priority 2: Analysis charts
```

**Impact:** Authors see authentic research results first when drafting sections.

### 2. Smart Filtering

**Automatically excludes:**
- Thumbnails, icons, logos, badges
- Build artifacts (node_modules/, __pycache__)
- Oversized files (>10MB figures, >5MB tables)
- Version control (.git/, .svn/)

**Impact:** Only relevant figures extracted, reducing noise.

### 3. Script Detection

Links figures to generating code:
```json
{
  "id": "fig_repo_003",
  "path": "figures/from_repo/results.png",
  "generating_script": "experiments/plot_results.py"
}
```

**Impact:** Enables reproducibility and verification.

### 4. Section Recommendations

Manifests suggest where figures belong:
```json
{
  "recommended_sections": ["methods", "results"]
}
```

**Impact:** Helps authors discover relevant figures during drafting.

### 5. Automatic README Indexes

Generated index for easy browsing:
```markdown
### fig_repo_001: Sensitivity plot
- **File**: `sensitivity_plot.png`
- **Original**: `manuscript/MicroGrowAgents_v6/figures/sensitivity_plot.png`
- **Sections**: results
- **Script**: `experiments/sensitivity_analysis.py`
```

**Impact:** Quick overview without parsing JSON.

## 🚀 Usage Examples

### Quick Start

```bash
# Extract figures and tables
/rrwrite-extract-figures-tables \
    --target_dir manuscript/project_v1 \
    --repo_path /path/to/analyzed/repo
```

### During Section Drafting

```python

# Get figures for results section (Priority 1 first)
```

## 🎓 Design Decisions

### Why Priority-Based?

**Problem:** Previous system mixed repository figures with generated charts, making it hard to identify actual research outputs.

**Solution:** Two-tier priority system with separate directories.

**Benefit:** Authors can focus on authentic results, use generated figures as supplements.

### Why Manifests?

**Problem:** File listings don't capture metadata (section relevance, generating scripts).

**Solution:** JSON manifests with rich metadata.

**Benefit:** Programmatic queries, section recommendations, script traceability.

### Why Script Detection?

**Problem:** Hard to verify figure authenticity and reproducibility.

**Solution:** Parse scripts in same directory for filename references.

**Benefit:** Links figures to code, enables verification.

### Why Separate Directories?

**Problem:** Mixing sources makes priority unclear.

**Solution:** `from_repo/` vs `generated/` subdirectories.

**Benefit:** Visual distinction, prevents confusion.
